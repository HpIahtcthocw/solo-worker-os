# Solo Worker OS

> AI-native freelance operating system. An AI agent manages your entire business workflow — client intake, project tracking, contract generation, payment follow-ups — through natural conversation.

Built for the **Qwen Cloud Global AI Hackathon Series** — Track 3: Expert Agent.

Powered by **Qwen Cloud** (DashScope API) with persistent memory via Supabase.

---

## What It Does

Solo Worker OS turns freelance business management into a conversation. Instead of juggling spreadsheets, docs, and invoicing tools, you just talk to the AI agent:

- **"New client — Marcus Rivera, e-commerce platform, budget $15,000, deadline August 15"** → Agent creates the project, generates a contract, and sets up deadline tracking
- **"Which projects are overdue?"** → Agent queries the database and prioritizes your follow-ups
- **"Write a follow-up email for the unpaid invoice from Sarah"** → Agent drafts a professional reminder in your chosen tone (polite / firm / final notice)

The agent runs a full agentic loop: it reasons over your request, selects the right tools, chains multi-step workflows, and executes autonomously.

### Key Capabilities

| Feature | How It Works |
|---------|-------------|
| **Multi-Tool Agent** | Qwen Cloud drives a reasoning loop with 7+ business tools (create project, update status, generate contract, generate follow-up, query knowledge base, etc.) |
| **Persistent Memory** | Every client, project, contract, and agent action is stored in Supabase PostgreSQL and injected into every conversation context |
| **Streaming Responses** | Real-time SSE streaming — see the agent's reasoning as it happens |
| **Bilingual UI** | One-click toggle between English and Chinese (中文) |
| **Automated Reports** | Cron-based weekly business summaries generated by Qwen |
| **Knowledge Base** | RAG-powered document storage for contracts, templates, and notes |
| **CSV Export** | Export project data for accounting and reporting |

## Tech Stack

| Layer | Technology |
|-------|-----------|
| **AI Agent** | Qwen Cloud (DashScope API) — streaming + function calling |
| **Framework** | Next.js 14 (App Router) + TypeScript |
| **Database** | Supabase (PostgreSQL) with Row Level Security |
| **Auth** | Supabase Auth (OTP / magic link) |
| **Styling** | Tailwind CSS + glassmorphism design system |
| **Deployment** | Vercel |
| **Email** | Resend (deadline reminders + weekly reports) |

### Multi-Agent Architecture

The agent layer is provider-agnostic. Qwen Cloud is the primary provider, with pluggable support for Claude and OpenAI:

```
User Message
    ↓
Agent Factory (lib/agent/index.ts)
    ↓
Qwen Provider (lib/agent/qwen.ts)
    ↓
Tool Loop (MAX_ITERATIONS = 6)
    ├── create_project
    ├── update_project_status
    ├── generate_contract
    ├── generate_followup_message
    ├── search_knowledge
    └── list_projects
    ↓
Streaming Response (SSE)
```

## Quick Start

### Prerequisites

- Node.js 18+
- A Supabase project (free tier works)
- A Qwen Cloud / DashScope API key

### Setup

1. **Clone and install:**

```bash
git clone <repo-url>
cd solo-worker-os
npm install
```

2. **Configure environment:**

```bash
cp .env.example .env.local
```

Edit `.env.local` with your keys:

```env
# Qwen Cloud (DashScope)
DASHSCOPE_API_KEY=your_dashscope_api_key
QWEN_MODEL=qwen-plus
AGENT_PROVIDER=qwen

# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# Optional: Email notifications
RESEND_API_KEY=your_resend_key
NOTIFICATION_EMAIL=you@example.com
```

3. **Set up the database:**

Run the SQL in `supabase/schema.sql` in your Supabase SQL Editor.

4. **Run the dev server:**

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### Demo Mode

The app runs in demo mode by default — no login required. This lets judges and testers explore all features immediately. To enable authentication, uncomment the `auth.getUser()` call in `middleware.ts`.

## Agent Tools

The agent has access to 7 business tools that it can call autonomously:

| Tool | Purpose | Example Trigger |
|------|---------|----------------|
| `create_project` | Create a new project with client details | "New client Sarah, logo design, $800, 3 weeks" |
| `update_project_status` | Change project status (active → delivered → invoiced → paid) | "Mark the Acme project as delivered" |
| `generate_contract` | Generate a full contract from project data | "Generate a contract for Marcus's project" |
| `generate_followup_message` | Draft payment reminder in selectable tone | "Write a firm follow-up for the unpaid Sarah invoice" |
| `search_knowledge` | Search the knowledge base (RAG) | "Do we have a template for NDA agreements?" |
| `list_projects` | List all projects with filters | "Show me all overdue projects" |
| `get_project_details` | Get full details of a specific project | "What's the status of the Marcus project?" |
